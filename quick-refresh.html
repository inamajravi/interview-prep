01 Design Patterns & SOLID

SRPOne reason to change. Watch for "and" in a class description.
OCPExtend without editing tested code.
LSPSubclass must be safely swappable for its base.
ISPMany small interfaces, not one fat one.
DIPDepend on abstractions — this is why DI containers exist.
StrategyMy real example: 3 matchers, one interface, weighted combo.
SingletonUsually overused — prefer container-managed AddSingleton.
Factory/BuilderHide construction complexity from the caller.

02 Architecture

MicroservicesIndependent deploy/scale per team — skip it below team-scale.
CQRSSplit read/write models — doesn't require two databases.
Event-DrivenPublish facts, don't call people — idempotency handles duplicates.
SagaChain of local transactions + explicit compensating "undo" steps.
DDDBusiness language in code; Bounded Contexts seed microservice boundaries.
Choreography vs OrchestrationOrchestrate once steps exceed ~4, for observability.

03 .NET Core & Spring Java

DI lifetime bugCaptive dependency — Scoped injected into Singleton gets pinned app-wide.
Spring default scopeSingleton by default — opposite instinct from .NET's per-request norm.
Async/awaitFrees the thread during I/O wait — throughput, not raw speed.
N+1 fix.Include() / JOIN FETCH — eager load instead of looping + lazy load.
Middleware orderException handler outermost; Auth before Authz.
@TransactionalAOP proxy — same idea as wrapping a using-block transaction.
JpaRepositoryQuery generated from method name — no implementation needed.
Minimal APIsGood for small focused services; Controllers scale better for large surfaces.

04 Angular

OnPush bugMutating in place ≠ new reference — reassign, don't mutate.
SignalsExplicit dependency tracking — no Zone.js tree-wide check needed.
switchMapTypeahead — cancel stale request on new input.
exhaustMapIgnore new triggers while one's in flight (login button).
concatMapQueue in order, one completes before next starts.
StandaloneSame DI/component model, just less NgModule bookkeeping.
StateBehaviorSubject for simple, NgRx once cross-team traceability matters.
Memory leakUnsubscribed .subscribe() — use async pipe or takeUntilDestroyed.

05 JUnit & Mockito

[Fact] → @Test. [Theory] → @ParameterizedTest.
Moq Setup/Verify → Mockito when(...).thenReturn(...) / verify(...)
@WebMvcTestLoads only the web layer — fast, most tests should live here.
@DataJpaTestRepository layer only, against test DB, auto-rollback.
TestcontainersReal Postgres/Kafka in Docker — closes the "works on H2, fails in prod" gap.
Over-mockingMock at real seams (external deps), not everything you own.

06 AWS & Azure

Horizontal scalingRequires statelessness — externalize session state first.
Lambda / FunctionsGreat for bursty/event work — bad at sustained high throughput (cost) and latency-sensitive work (cold starts).
ALB vs NLBContent-aware routing? ALB (L7). Raw speed/TCP? NLB (L4).
SQS vs SNSOne consumer (queue) vs fan-out to all subscribers (topic).
AKS/EKSManaged control plane — skip unless you truly need K8s flexibility.
Service BusSQS+SNS combined, plus ordered sessions and dedup built in.
APIMCentralize auth/throttling/versioning instead of duplicating per service.

07 Agentic AI / Copilot / MCP

Agentic vs chatbotPlans, calls tools, observes results, loops — vs single Q&A.
MCPStandard protocol so any AI client calls any tool — N×M → N+M integrations.
Best prompting habitAsk for a plan before code on non-trivial tasks.
Your framingArchitect/reviewer — same skill as reviewing a junior dev's PR.

08 Message Queues

Why a queue at allDecouples producer/consumer — producer's job ends at "safely enqueued."
Delivery guaranteeAt-least-once is the real-world default — pair with idempotency, don't rely on the broker for exactly-once.
IdempotencyCheck a transaction ID against a persisted record before acting — makes duplicates harmless.
Visibility timeoutMessage re-appears if a consumer dies mid-processing without deleting it.
DLQStops a permanently-broken message from retrying forever — surfaces it instead.
SQS vs SNSOne consumer (queue) vs fan-out to all subscribers (topic) — combine via SNS→SQS fan-out.
FIFOUse only when order matters for a specific entity (e.g. same account) — not a default.
